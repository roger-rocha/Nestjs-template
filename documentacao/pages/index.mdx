## Como está estruturado essa documentação

### Ela está dividida em 3 partes:

1. Partes comuns de todo o projeto

2. Partes específicas para a parte do cliente do projeto

3. Partes específicas para a parte do administrador do projeto

## Por que essa documentação existe?

- Para facilitar a vida de quem está entrando no projeto

- Por que eu queria ter tido uma documentação assim quando entrei no projeto

- Por que eu quero que o projeto continue sendo desenvolvido mesmo depois que eu sair

## Como contribuir com essa documentação

- Só ler a documentação do nextra e seguir os passos https://nextra.site/docs

- Ir colocando aqui tudo que você achar que é importante para o projeto

- Colocar aqui toda nova mudança de regra de negócio que for feita

- Colocar aqui toda nova implementação, para que você lembre mais tarde ou para o próximo desenvolvedor saber o que foi feito
